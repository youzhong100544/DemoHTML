接下来，我们得想办法画一个圆圈，这个圆圈用什么画那？
你肯定不能用 border，不然的话，你一放大，这个 border 会跟着放大。

我们得用啥那？
这里想到的办法是使用径向渐变 radial-gradient()；

我们使用 background: radial-gradient()；
先给它扩散一个黄色的内容，然后在扩散一个红色的边框；

但是我们先思考一个问题，就是对这个圆，有没有限制，
因为默认情况下。圆扩散成 100% 的时候，它的圆是把这个边框包围起来的，
而我们希望的是 圆扩散成 100% 的时候 在边框里面；
因此我们要限制一下这个圆的扩散范围，
怎么限制那？
我们可以使用 closest-side， 表示以最短边做为圆半径，
那么之后写颜色的时候， 百分比就相对于这个最短边的。
颜色使用黄色，

那颜色扩散多少那？
我们来研究一下，
应该是扩散到 100% 减去这个边框值，
比如边框的粗细是 5px，那么就是 100% 减去 5px，来填充黄色

那么后续的颜色是啥？
接下来就是红色的边框，
它要从黄色边缘扩散到 100%





那么之后颜色是啥？
之后就随便用一个颜色填充
我们这里用蓝色





现在还有一个小问题，就是这个边框有锯齿
这是渐变的老问题了，

解决方式呢，基本上都是把两个颜色之间 相隔 那么一点点距离





接下来就是，鼠标移入，它会放大
但是你放大归放大，
图片头像可以放大，但是这个背景图（黄色背景、红色边框）他要保持原始尺寸

纵向的尺寸先不管；
横向的尺寸，我要始终保持这个背景图的尺寸看上去是不变的，即便整个元素放大了
那么怎么做那？
如果元素放大了两倍，那么我这个背景图就要缩减为原来的 1/2
也就是说这里的背景图的横向尺寸，应该是100%除以放大倍数，也就是 calc(100% / 放大倍数)
纵向的尺寸 为 100%；

尺寸的代码就是
横向：calc(100% / 放大倍数)
纵向：100%

background：no-repeat center/calc(100% / var(--scale_factor)) 100%;





现在我们还要解决一个问题，
因为这是个背景，就导致了这个图片内容，它是覆盖背景的；
覆盖背景倒是没事问题，关键是我们要怎么把下面的边框红线显示出来？

我们就可以借助 outline





然后我们可以使用 border-radius 定形

左上角不变、右上角不变，
左下角不变、右下角给它设打一点；
border-radius: 0 0 999px 999px;

这样 outline 就显示为 border-radius 的形状了





现在外边框离它的内容之间呢有点间隙
这个我们使用 outline-offset，
outline-offset 表示外边框和元素之间的距离
你把这个距离设的越大，那他的距离越远

它也可以为负数

最好还是用变量的方式





现在问题又来了，
我们需要搞定的问题是，当鼠标移入的时候这图片放大
但是外边框的尺寸保持不变
也就是说，我们根据目前的信息，
动态的调整这个 outline-offset 的值

那怎么调整呢？

现在，这个外边框是这个样子的，
图片被放大了 2 倍
那么外边框自然而然也会跟着放大 2 倍
现在我们要让这个外边框往回缩

缩多少呢？

是不是等于
（放大过后的宽度 - 放大之前的宽度） / 2
也就是
（原始的宽度 * 放大倍率 - 原始的宽度）/ 2

但是，由于我们这个 outline-offset 它设置的值是相对于原始尺寸的
也就是说，原始的宽度为200，
那 outline-offset 的值 最多只能是 -100

那如果说放大了 1000 倍，
那（原始的宽度 * 放大倍率 - 原始的宽度）/ 2 的结果是不是远远超过 100 了

就是你算出来的这个值，是被放大过后的值，
你要得到原始的收缩的话，还要去除以一个放大倍率

也就是
（（原始的宽度 * 放大倍率 - 原始的宽度）/ 2） / 放大倍率

数学换算得到
（原始的宽度 - 原始的宽度 / 放大倍率 ）/ 2

但是 （原始的宽度 - 原始的宽度 / 放大倍率 ）得到的是正值
我们需要往里收缩，需要一个负值

所以把它反过来相减就可以了
就是
（原始的宽度 / 放大倍率 - 原始的宽度）/ 2

结果就是把
outline-offset: calc(
 0px - var(--width_circle_border)
);
替换成
outline-offset: calc(
 ((var(--size)/var(--scale_factor) - var(--size)) / 2) - var(--width_circle_border)
);
